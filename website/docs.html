ScamShield ← Back to app

Documentation

A rule-based, explainable scam-risk scorer for job/internship offers, plus an AI resume-to-job matcher. Built as a Chrome extension + website sharing one backend.

Tech stack

Node.js + Express MongoDB Atlas Manifest V3 (Chrome) Gemini API — free tier RDAP — free, no key Google Custom Search — free tier Docker GitHub Actions CI/CD

Architecture

One Express backend serves three clients: the Chrome extension popup, this website, and (indirectly) MongoDB for persistence. Every risk score is explainable — a weighted, rule-based scorer, not a black-box model — so every point can be traced to a specific finding.

Offer text
   │
   ├─► Regex extraction (company, role, stipend, red flags)
   │      │
   │      └─► low confidence? ─► Gemini fallback (structured JSON)
   │
   ├─► Domain age check (RDAP)
   ├─► LinkedIn presence check (Google Custom Search)
   ├─► Community reports (MongoDB)
   ├─► Reddit mentions (mocked - see note below)
   │
   └─► Weighted scorer ─► 0-100 risk score + itemized findings

API reference

POST /api/analyze

Analyzes a job/internship offer for scam risk.

// Request
{ "text": "Full offer email or message text..." }

// Response
{
  "extracted": {
    "company": "BrightFuture Info Pvt Ltd",
    "role": "Data Entry Intern",
    "stipend": "45000 per month",
    "eligibility": null,
    "emailDomain": "gmail.com"
  },
  "extractionMethod": "regex" | "regex+llm",
  "riskScore": 80,
  "riskLevel": "low" | "medium" | "high",
  "findings": [{ "severity": "bad" | "ok", "message": "..." }],
  "mentions": [{ "title": "...", "url": "..." }]
}

POST /api/report

Submits a user report about a company - factored into future risk scores for that company.

// Request
{ "companyName": "BrightFuture Info Pvt Ltd", "reason": "Asked for a fee, never responded after paying." }

// Response
{ "success": true, "totalReports": 3 }

POST /api/analyze-resume

Scores how well a resume matches a job description. Requires GEMINI_API_KEY - no regex fallback exists for this feature.

// Request
{ "resumeText": "...", "jobDescriptionText": "..." }

// Response
{
  "matchScore": 82,
  "summary": "...",
  "matchedSkills": ["..."],
  "missingSkills": ["..."],
  "suggestions": ["..."]
}

Scoring model

Each signal adds a fixed or capped point value to a 0-100 score, with a human-readable finding attached to every point. This is a deliberate choice over a trained ML model: every score is traceable to a specific, explainable reason.

SignalMax points
Free email domain30
Red-flag phrases (fees, urgency, etc.)15 each
Missing stipend / role10-15
Scam mentions found online30
User-submitted reports25
Domain registered <30 days ago25
No LinkedIn presence found15

A note on data sources

Reddit tightened non-commercial API approval mid-project, so redditSearch.js currently returns realistic mock data instead of live results. The real OAuth implementation is preserved in redditSearch.live.js and documented as a one-file swap once access is approved. See the GitHub repo README for full details.

Why this matters

Recent survey data shows this isn't a niche problem: 93% of Indian job seekers report encountering fraudulent postings, and Gen Z respondents were disproportionately affected - 46% reported losing money to recruitment scams. Source: Indeed survey via Business Standard, July 2026
